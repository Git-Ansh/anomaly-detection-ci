% ABSTRACT
% ============================================================
\begin{abstract}
Machine learning classifiers for software bug triage produce point predictions without reliability guarantees. We apply conformal prediction to multi-class bug component assignment, to the best of our knowledge, the first such application in software engineering. We introduce MACCP (Model-Agreement Conditioned Conformal Prediction), which partitions bugs by whether two diverse classifiers (a DeBERTa transformer and an XGBoost tree ensemble) agree on the predicted component. On Eclipse (30 components, 158K resolved bugs), the two models agree on 46.7\% of test bugs with 85.7\% accuracy when they agree versus 27.1\% when they disagree. Within the same DeBERTa confidence bins, agreement provides 43--69 percentage point accuracy gaps on Eclipse and 12--56 percentage points on Mozilla Core, a signal largely independent of single-model confidence. MACCP exploits this: on Eclipse, the agree partition achieves 61.2\% single-component sets with mean set size 2.30, allowing automated assignment of 29\% of bugs at 90\% singleton accuracy. On Mozilla Core, the agree partition (30.9\% of bugs) achieves 14.1\% singletons at 99.2\% accuracy ($\alpha{=}0.20$). We evaluate on three datasets (Eclipse, Mozilla Core, ServiceNow) and demonstrate that conformal coverage guarantees hold within 1--4 percentage points of nominal levels despite temporal distribution shift spanning 15--23 years.
\end{abstract}

Conformal prediction, bug triage, prediction sets, cross-model agreement, software quality

% I. INTRODUCTION

\section{Introduction}\label{sec:intro}

In large-scale software projects, incoming bug reports must be assigned to the correct development team for resolution, a process called bug triage. Eclipse, an open-source IDE with over 300{,}000 filed bug reports~\cite{lamkanfi2013eclipse}, distributes work across 30 component teams. Misrouting is common and costly: Jeong et al.~\cite{jeong2009improving} found that 37--44\% of bugs in Mozilla and Eclipse undergo at least one reassignment, each adding days to the resolution timeline.

Machine learning can automate this decision, and current approaches~\cite{anvik2006should,xia2017elm,dipongkor2023transformer,chen2025triangle,xu2026whatscode} achieve reasonable accuracy. But accuracy alone does not identify \emph{which} predictions are trustworthy. A classifier that is 66\% accurate overall provides no mechanism to distinguish the 66\% it gets right from the 34\% it gets wrong. The standard proxy -- softmax confidence -- is the implicit reliability signal in every deployed triage system: high-confidence predictions are trusted, low-confidence ones are escalated. We test this assumption directly and find it insufficient: within the highest DeBERTa confidence bin (0.85--1.0), accuracy varies by 43 percentage points depending on whether an independent model corroborates the prediction (Table~\ref{tab:agreement-gaps}). Confidence measures how strongly a model believes its own answer. It does not measure whether that answer is correct.

Conformal prediction~\cite{vovk2005algorithmic} addresses this limitation by producing prediction sets rather than point predictions. A conformal set contains the correct component with probability $\geq 1 - \alpha$, providing a formal coverage guarantee. When the set contains a single component, the assignment can be automated with quantified confidence. When it contains multiple components, the developer selects from a reduced shortlist rather than scanning all 30 options.

Bug difficulty varies widely -- some reports clearly implicate a single team through a specific module crash and stack trace, while others are genuinely ambiguous. We observe that when two very different AI models (one that reads the bug report's text (DeBERTa~\cite{he2021deberta}) and one that analyzes metadata like who filed the bug and which product it belongs to (XGBoost~\cite{chen2016xgboost})) independently point to the same team, the bug is almost certainly assigned correctly (85.7\% accuracy on Eclipse). When they disagree, the prediction is unreliable (27.1\% accuracy). We use this agreement signal to adjust the system's shortlists: tighter for easy bugs, wider for hard bugs.

We evaluate on Eclipse (30 components, 158K bugs), Mozilla Core (20 components, 45.7K bugs), and ServiceNow (11 IT incident categories, 22.7K incidents). On Eclipse, the agree partition (46.7\% of bugs) achieves 61.2\% singleton rate at 90.0\% accuracy with mean set size 2.30, enabling automated triage for 29\% of all incoming bugs with formal coverage guarantees. On Mozilla Core, MACCP replicates: the agree partition (30.9\%) achieves 14.1\% singleton rate at 99.2\% accuracy ($\alpha{=}0.20$). Cross-model agreement provides 33--69 percentage point accuracy gaps within the same confidence bins across both datasets, a signal largely independent of single-model confidence.

In practice, when development teams already deploy multiple classifiers, as is increasingly common in production triage systems~\cite{chen2025triangle,xu2026whatscode}, cross-model agreement can be extracted at minimal additional cost (comparing two top-1 predictions) and used as a conditioning signal for uncertainty quantification with formal guarantees.

\smallskip\noindent\textbf{Contributions.}
\begin{enumerate}
\item \textbf{To the best of our knowledge, the first application of conformal prediction to multi-class SE triage.} A recent survey of automated bug triage~\cite{zhao2025triage} identifies no prior CP applications, and our own literature search confirms this gap. The only SE application is Shahini et al.~\cite{shahini2024conformal} for binary just-in-time defect prediction, a fundamentally different 2-class setting. We demonstrate that conformal prediction produces practically useful prediction sets for 30-class component assignment.

\item \textbf{Cross-model agreement as an independent reliability signal.} We show that agreement between architecturally diverse classifiers provides 43--69 percentage point accuracy improvements within identical confidence bins. This finding is consistent with ensemble diversity theory~\cite{kuncheva2003measures} but has not been previously quantified as a conditioning variable for conformal prediction.

\item \textbf{MACCP: a practical recipe for conditional conformal triage.} We instantiate the Gibbs et al.~\cite{gibbs2025conditional} conditional coverage framework with cross-model agreement as the grouping function. The agree partition (47\% of bugs) achieves 61.2\% singletons at 90\% accuracy, allowing automated assignment of 29\% of bugs.
\end{enumerate}

\noindent Together, these contributions demonstrate that single-model confidence is an insufficient basis for automating SE triage decisions, and that cross-model agreement provides the missing reliability signal.

\smallskip\noindent\textbf{Research Questions.}
\begin{itemize}
\item[\textbf{RQ1}] Is single-model confidence sufficient for reliability partitioning in bug triage, or does cross-model agreement provide information that confidence alone cannot capture?
\item[\textbf{RQ2}] Can we exploit this cross-model agreement to produce more efficient conformal prediction sets, specifically smaller shortlists when models agree and larger safety nets when they disagree?
\item[\textbf{RQ3}] Do the formal coverage guarantees survive when the software project evolves significantly between the training period and deployment, spanning 15 years?
\item[\textbf{RQ4}] Does this approach generalize across different software ecosystems with different component taxonomies and data characteristics?
\end{itemize}

\section{Background and Related Work}\label{sec:background}

\subsection{Conformal Prediction}\label{sec:bg-cp}

Train a model. Hold out a calibration set the model has never seen. For each calibration example, measure how ``surprised'' the model is by the true answer; this surprise is called a \emph{nonconformity score}. High surprise means the model found this example hard. Collect all the surprise scores and pick a threshold: the 90th percentile means ``be surprised enough to include the true answer 90\% of the time.'' At test time, include any class whose surprise score falls below the threshold. The result is a prediction set guaranteed to contain the correct answer with probability $\geq 1 - \alpha$. This probability, the fraction of test instances where the true label falls within the prediction set, is called the \emph{empirical coverage}.

Split conformal prediction~\cite{vovk2005algorithmic} provides distribution-free prediction sets with finite-sample coverage guarantees. Given a calibration set $\calD_\text{cal} = \{(x_i, y_i)\}_{i=1}^{n}$, a trained classifier $f$, and a nonconformity score function $s(x,y)$ measuring how poorly $y$ conforms to $f$'s prediction for $x$, the conformal threshold is:
\begin{equation}
\hat{q} = \Quantile\!\left(\frac{\lceil (n+1)(1-\alpha) \rceil}{n};\; \{s_i\}_{i=1}^{n}\right)
\end{equation}
The prediction set for a test point $x$ is $\calC(x) = \{y : s(x,y) \leq \hat{q}\}$, satisfying $\PP(Y \in \calC(X)) \geq 1 - \alpha$ under exchangeability of calibration and test data.

Adaptive Prediction Sets (APS)~\cite{romano2020classification} define the score as the cumulative probability mass needed to include the true class when iterating through classes in decreasing probability order. Regularized APS (RAPS)~\cite{angelopoulos2021uncertainty} adds a penalty $\lambda \cdot \max(0, j - k_\text{reg})$ for each class $j$ beyond rank $k_\text{reg}$, discouraging inclusion of low-ranked classes and producing 5--10$\times$ smaller sets than APS in practice.

Mondrian conformal prediction~\cite{vovk2005algorithmic} partitions the calibration data into groups and computes a separate conformal threshold per group, yielding per-group coverage guarantees: $\PP(Y \in \calC(X) \mid G(X) = g) \geq 1 - \alpha$ for each group $g$. Gibbs et al.~\cite{gibbs2025conditional} formalize the general framework for conditional coverage through grouping functions. MACCP is an instantiation of this framework.

When applied to distinct groups, Mondrian CP computes separate thresholds for each group. A group with higher base accuracy receives a tighter threshold, producing smaller sets. A group with lower base accuracy receives a wider threshold, producing larger sets. The coverage guarantee of $\geq 1 - \alpha$ holds independently within each group.

\subsection{Conditional Coverage and Related Methods}

Marginal coverage hides per-group miscoverage: easy instances are overcovered (unnecessarily large sets) and hard instances are undercovered (dangerously small sets). Barber et al.~\cite{barber2021impossibility} prove that exact conditional coverage is impossible without assumptions, motivating approximate conditioning via grouping.

Kaur et al.~\cite{kaur2025trust} condition conformal prediction on trust scores, kNN distance ratios measuring deviation from Bayes optimality. Trust scores are a continuous signal requiring kNN computation and hyperparameter selection (number of neighbors, distance metric). MACCP differs: a binary signal from a trained second model, no hyperparameters, and direct mapping to practitioner workflows (auto-assign vs.\ escalate). The 43--69$\pp$ gaps demonstrate that the binary signal is strong enough.

Alami et al.~\cite{alami2025sacp} symmetrically aggregate conformity scores from multiple models. Ochoa Rivera et al.~\cite{ochoarivera2025csa} improve ensemble conformal prediction efficiency via score-based aggregation. Luo and Zhou~\cite{luo2025optimal} propose weighted aggregation of conformity scores for classification. These methods use richer information from auxiliary models at the cost of complexity. MACCP trades information content for simplicity and interpretability, reducing a full model's output to a single bit. The empirical results demonstrate that this binary signal suffices for SE triage.

Murphy~\cite{murphy2026calpoly} independently explores ensemble disagreement for conformal prediction under distribution shift. That work focuses on theoretical properties of disagreement-based partitioning; we provide the first large-scale empirical evaluation on real-world SE triage tasks.

\subsection{Bug Triage and SE Classification}

Cubranic and Murphy~\cite{cubranic2004automatic} first applied text categorization to developer assignment. Anvik et al.~\cite{anvik2006should} established SVM and Naive Bayes baselines. Jeong et al.~\cite{jeong2009improving} quantified the cost of misrouting, finding that 37--44\% of bugs undergo reassignment. Bhattacharya and Neamtiu~\cite{bhattacharya2010fine} introduced incremental learning and tossing graphs. Deep learning~\cite{xia2017elm} and transformer architectures~\cite{dipongkor2023transformer} followed. Recent industrial systems -- Microsoft's Triangle~\cite{chen2025triangle} and Meta's WhatsCode~\cite{xu2026whatscode} -- leverage LLMs for incident and bug triage. These approaches produce point predictions without reliability guarantees, leaving practitioners unable to distinguish correct from incorrect assignments before routing.

Zhao et al.~\cite{zhao2025triage} survey automated bug triage and identify no conformal prediction applications. Shahini et al.~\cite{shahini2024conformal} present the only prior CP application in SE: binary just-in-time defect prediction, where sets contain at most 2 classes. Shahini et al.~\cite{shahini2025miscalibrated} further show that SE classifiers exhibit expected calibration error of 2--35\%, motivating distribution-free alternatives.

Temporal distribution shift is persistent in SE: component vocabularies evolve and APIs are deprecated. Grinsztajn et al.~\cite{grinsztajn2022tree} show tree-based models outperform deep learning on tabular data. We extend this finding to temporal shift settings where metadata features prove more stable than text.

\subsection{Base Classifiers}

\textbf{DeBERTa-v3-base}~\cite{he2021deberta} is a 184M-parameter transformer fine-tuned on bug report text (Summary~\texttt{[SEP]}~Description, truncated to 1500 characters). The disentangled attention mechanism processes contextual token embeddings through self-attention, and a classification head maps the pooled output to $K$ classes.

\textbf{XGBoost}~\cite{chen2016xgboost} processes TF-IDF features (500 dimensions from text) concatenated with engineered metadata: product encoding, creator bug count, severity, priority, operating system, platform, and component size computed from training data. For Mozilla Core, we additionally include reporter component history profiles (the distribution of components a reporter has previously filed bugs in). We use 100 estimators with max depth 6.

The choice of these two specific models is deliberate. DeBERTa captures what the bug report \emph{says}: the semantic content of crash descriptions, error messages, and reproduction steps. XGBoost captures the \emph{context} around the report: who filed it (reporter identity correlates with component assignment, as individual developers tend to work within specific subsystems), what product it belongs to (product strongly predicts component), and what severity was assigned. These are fundamentally different information channels. A crash report's text might mention ``rendering'' (suggesting the UI team), but the reporter might be a Platform team developer who always files platform bugs (suggesting Platform). When both channels point to the same answer, the evidence is strong. When they point to different answers, the bug is genuinely ambiguous. This architectural diversity is what makes the agreement signal informative; two models that process data identically would agree or disagree for the same reasons, providing no additional signal.

\subsection{MACCP Formulation}\label{sec:maccp-formulation}

We describe the procedure in four steps, then formalize each mathematically. Step~1: check if both models predict the same class. Step~2: split the calibration set into ``agree'' bugs and ``disagree'' bugs. Step~3: compute a separate conformal threshold for each group. Step~4: at test time, use the tight threshold when models agree, the wide threshold when they disagree. The equations below formalize these four steps.

The nonconformity score (Equation~\ref{eq:raps-score}) formalizes the intuitive notion of ``surprise.'' For each calibration bug, we sort the model's predicted class probabilities from highest to lowest and accumulate them until the true component is reached. If the true component is the model's top prediction, very little probability mass is needed: low surprise, low score. If the true component is ranked 15th out of 30, substantial probability mass must be accumulated: high surprise, high score. RAPS adds a small penalty for each class included beyond rank $k_\text{reg}$, discouraging the inclusion of low-probability classes and producing tighter sets.

Let $f_1$ (DeBERTa) produce a probability distribution $\mathbf{p}_1(x) = (p_1^{(1)}(x), \ldots, p_1^{(K)}(x))$ over $K$ classes, and let $f_2$ (XGBoost) produce top-1 prediction $\hat{y}_2(x)$.

\smallskip
\noindent\textbf{Step 1: Agreement indicator.}
\begin{equation}\label{eq:agreement}
A(x) = \indicator{\argmax_k\, p_1^{(k)}(x) \;=\; \hat{y}_2(x)}
\end{equation}

\smallskip
\noindent\textbf{Step 2: Partition the calibration set.}
Given $\calD_\text{cal} = \{(x_i, y_i)\}_{i=1}^{n}$:
\begin{align}
\calD_\text{agree} &= \{(x_i, y_i) \in \calD_\text{cal} : A(x_i) = 1\} \\
\calD_\text{disagree} &= \{(x_i, y_i) \in \calD_\text{cal} : A(x_i) = 0\}
\end{align}

\smallskip
\noindent\textbf{Step 3: Compute RAPS nonconformity scores} on $f_1$'s calibration softmax probabilities. Let $\pi_{(j)}(x)$ denote the $j$-th largest softmax probability and $k_i$ the rank of the true class $y_i$:
\begin{equation}\label{eq:raps-score}
s_i = \sum_{j=1}^{k_i} \Big(\pi_{(j)}(x_i) + \lambda \cdot \max(0,\, j - k_\text{reg})\Big)
\end{equation}
with $\lambda = 0.01$ and $k_\text{reg} = 5$.

\smallskip
\noindent\textbf{Step 4: Group-specific quantiles.}
For each group $g \in \{\text{agree}, \text{disagree}\}$:
\begin{equation}\label{eq:quantile}
\hat{q}_g = \Quantile\!\left(\frac{\lceil(|\calD_g|+1)(1-\alpha)\rceil}{|\calD_g|};\; \{s_i : i \in \calD_g\}\right)
\end{equation}

\smallskip
\noindent\textbf{Step 5: Prediction sets at test time.}
For a new instance $x$:
\begin{equation}\label{eq:pred-set}
\calC(x) = \left\{y : \sum_{j=1}^{\text{rank}(y)} \!\Big(\pi_{(j)}(x) + \lambda \cdot \max(0,\, j - k_\text{reg})\Big) \leq \hat{q}_{A(x)}\right\}
\end{equation}

\smallskip
\noindent\textbf{Coverage guarantee.}
From Mondrian CP theory~\cite{vovk2005algorithmic}:
\begin{equation}\label{eq:coverage}
\PP\!\big(Y \in \calC(X) \mid A(X) = a\big) \geq 1 - \alpha \quad \text{for each } a \in \{0, 1\}
\end{equation}

The formulation instantiates the Gibbs et al.~\cite{gibbs2025conditional} conditional coverage framework with a binary grouping function; the theoretical guarantee holds under exchangeability within each group.

\smallskip\noindent\textbf{Worked example.} A bug with Summary ``NPE in org.eclipse.debug.core.DebugPlugin'' is processed by both models. DeBERTa assigns 72\% probability to Debug and 15\% to Platform. XGBoost also predicts Debug. Since both agree, the system uses the agree threshold ($\hat{q}_{\text{agree}} = 0.953$ at $\alpha{=}0.10$). Accumulating DeBERTa's ranked probabilities -- Debug (0.72), Platform (0.15), Core (0.06) -- reaches 0.93, below the agree threshold of 0.953. Adding UI (0.04) exceeds it. The set \{Debug, Platform, Core\} contains three components with a 90\% coverage guarantee.

\subsection{Data Preparation and Temporal Split}

We use a strictly chronological split: 60\% train (oldest), 20\% calibration (middle), 20\% test (newest). Components are selected as the top-$K$ by training-set frequency. Components outside the top-$K$ are excluded, following Anvik~\cite{anvik2006should}: catch-all classes lack cohesive feature distributions and inflate nonconformity score variance during calibration, producing artificially wide prediction sets. In a deployed system, bugs whose maximum predicted probability across all $K$ known components falls below a configurable threshold would bypass the conformal prediction pipeline entirely and route directly to human triage, functioning as an implicit out-of-vocabulary detector. Noise filtering removes INVALID, DUPLICATE, WONTFIX, and WORKSFORME resolutions, standard in Eclipse triage literature~\cite{lamkanfi2013eclipse,dipongkor2023transformer}. Canonical label mapping is computed from training data only and shared across both models.

The temporal split violates strict exchangeability between calibration and test data. Following Barber et al.~\cite{barber2023conformal} and Oliveira et al.~\cite{oliveira2024split}, we note that split conformal prediction maintains approximate validity under mild distribution shift, with coverage degradation proportional to the total variation distance between calibration and test distributions. We report empirical coverage at all $\alpha$ levels to verify this holds in practice.

\subsection{Baselines and Metrics}

\textbf{Baselines:}
(1)~DeBERTa RAPS: standard split conformal on DeBERTa alone;
(2)~XGBoost RAPS: standard split conformal on XGBoost alone;
(3)~MACCP (proposed): Mondrian CP conditioned on agreement;
(4)~Softmax threshold baselines for selective prediction (non-conformal).

\textbf{Metrics:}
Empirical coverage (should be $\geq 1 - \alpha$);
mean and median prediction set size (smaller = more informative);
singleton rate (fraction of sets with exactly 1 class);
singleton accuracy (of singletons, fraction correct);
AUGRC (Area Under Generalized Risk-Coverage Curve~\cite{traub2024augrc}), a threshold-free measure of selective prediction quality where lower values indicate the classifier maintains lower error rates across wider coverage levels; with bootstrap 95\% CIs (10K resamples);
within-confidence-bin accuracy gaps (for RQ1).

\textbf{Alpha levels:} $\alpha \in \{0.01, 0.05, 0.10, 0.20\}$.

\smallskip
The magnitude of these gaps has a clear mechanistic explanation. DeBERTa's errors are primarily caused by temporal vocabulary drift: bug reports from 2014--2024 reference frameworks, APIs, and coding patterns absent from the 2001--2009 training data. When DeBERTa encounters unfamiliar terminology, it may still produce a confident prediction based on partial pattern matches, but the prediction is often wrong. XGBoost, relying on temporally stable metadata (product encoding, reporter identity, severity), is unaffected by vocabulary drift and may correctly predict a different component. Disagreement thus specifically flags instances where DeBERTa's confidence is miscalibrated. This explains why the agreement signal is not redundant with confidence: confidence measures how strongly the text model believes its own prediction, while disagreement measures whether an independent information channel corroborates that belief.

We note that while these gaps are large, they may partly reflect the specific characteristics of the Eclipse dataset, particularly the severity of the 15-year temporal drift and the strong predictive power of the product encoding feature. The substantial gaps observed on Mozilla Core (12--56$\pp$) confirm the phenomenon generalizes, though further investigation across additional datasets would strengthen this conclusion.

To reach 90\% accuracy on retained predictions, softmax thresholding requires DeBERTa confidence above 0.99 --- a bar so high that only 12.8\% of bugs pass it. Ensemble voting does better on aggregate accuracy (67.1\%) and the agree/disagree partition does identify reliable predictions (85.7\% vs.\ 50.7\%), but without conformal prediction the output is still a single class with no coverage guarantee for uncertain cases. Single-model RAPS addresses the guarantee gap by producing calibrated prediction sets, yet it applies a uniform threshold --- the same quantile for bugs a domain expert would route in seconds and bugs that would require an hour of investigation. MACCP resolves this tension. MACCP covers a larger fraction of bugs at similar singleton quality and requires no hyperparameter selection (number of neighbors, distance metric).

An earlier design explored multi-stage cascade architectures with confidence gating between sequential stages. We abandoned it after observing conditional independence between stages sharing text features: upstream predictions contributed no additional signal to downstream classifiers. Parallel cross-model agreement, drawing on fundamentally different feature channels, proved far more informative.

The explanation generator on Eclipse partitions 25,499 test bugs into operational categories: AUTO-TRIAGE 7,296 (28.6\%), AUTO-TRIAGE WITH CAUTION 20 (0.1\%), REVIEW SHORTLIST 2,525 (9.9\%), REVIEW NEEDED 1,367 (5.4\%), and NEEDS CAREFUL REVIEW 14,291 (56.0\%). The 28.6\% auto-triage rate with 90\% accuracy represents a substantial reduction in manual triage workload; the careful-review group still benefits from the search space reduction noted in Section~\ref{sec:rq2}.

\section{Discussion and Threats to Validity}\label{sec:discussion}

\subsection{Practical Implications}

As detailed in Section~\ref{sec:rq2} and Section~\ref{sec:ablation}, the framework partitions bugs into three operational tiers: full automation via agree-group singletons (29\% at 90\% accuracy), shortlist review via agree-group non-singletons (10\%, mean set size 2.30), and careful review with uncertainty context via the disagree group (56\%, mean set size 8.52). The 10\% error rate among auto-triaged singletons compares favorably to the 37--44\% bug reassignment rate reported by Jeong et al.~\cite{jeong2009improving}, though we note that reassignment is not synonymous with initial misassignment --- some reassignments reflect organizational changes rather than routing errors. The practical implication is that triage systems relying solely on confidence thresholds for selective automation will misidentify a substantial fraction of unreliable predictions as trustworthy --- a failure mode that cross-model agreement detects.

\subsection{Cost-Benefit Analysis}

Maintaining two diverse classifiers has engineering overhead, and whether MACCP's 29\% automation rate justifies that cost depends on triage volume. With XGBoost RAPS alone, 20.1\% of predictions are singletons at 93.6\% accuracy. MACCP adds 9 percentage points of automation (29\% vs 20\%) at the cost of deploying a transformer model.

At Eclipse's scale of roughly 100 bugs per day, DeBERTa inference totals about 5 seconds of daily GPU time --- absorbed within spare cycles on any commodity cloud GPU instance. At enterprise scale (1,000 bugs/day), the 290 daily auto-triaged bugs each eliminate a manual classification decision, and since triage of a 30-component bug report typically involves reading the report, consulting team ownership documentation, and making a routing judgment, the aggregate time savings at enterprise volume quickly dominate infrastructure costs. Below a certain volume threshold that depends on local GPU costs and engineer salaries, single-model XGBoost RAPS becomes the more practical choice --- a tradeoff we arrived at after evaluating deployment scenarios, which led us to position XGBoost RAPS as a GPU-free alternative with modestly lower automation (20.1\% vs 29\% singletons). Batch processing workflows, common in bug triage where reports accumulate in queues, eliminate per-prediction latency concerns entirely; DeBERTa inference runs on the order of tens of milliseconds per prediction on modern GPU hardware.

The conformal prediction and agreement conditioning mechanisms are model-agnostic: they apply to any pair of diverse classifiers, including large language models. An LLM-based text classifier would plausibly achieve higher base accuracy than DeBERTa, though the magnitude depends on the specific LLM's exposure to software engineering terminology during pretraining and its behavior under temporal domain shift --- factors we have not evaluated. Higher base accuracy would mechanically produce tighter conformal sets and higher singleton rates; the coverage guarantee holds regardless. The choice of base classifiers represents an accuracy-cost tradeoff orthogonal to MACCP itself.

These results are specific to open-source Bugzilla-based projects with rich metadata. Whether the agreement signal provides similar value in commercial settings with different metadata structures (e.g., Jira, Azure DevOps) remains untested.

\subsection{Why Metadata XGBoost Outperforms DeBERTa}

XGBoost outperforms DeBERTa on both datasets: 66.0\% versus 54.5\% on Eclipse and 50.4\% versus 39.1\% on Mozilla Core. Metadata features --- product encoding, reporter identity --- carry the same signal in 2005 and 2020; text vocabulary does not.

\subsection{Threats to Validity}

\textbf{Internal.}
(1)~The temporal split violates strict exchangeability. We concede this explicitly and report empirical coverage to verify approximate validity.
(2)~Label noise in Eclipse data: bugs may have been reassigned after initial triage; we use the final component as ground truth.
(3)~``Other'' class exclusion: we report both 30-class (excluding Other) and 31-class (including Other) results. Exclusion follows Anvik~\cite{anvik2006should} and is necessary for conformal calibration; catch-all classes lack cohesive feature distributions.

\textbf{External.}
(1)~MACCP requires two competent models with diverse feature representations. It fails when one model is too weak.
(2)~Eclipse and Mozilla Core are both Bugzilla-based open-source projects. Generalization to commercial bug trackers (Jira, Azure DevOps) with different taxonomies is untested.
(3)~The framework assumes a fixed component taxonomy. Component creation or deprecation during deployment requires periodic recalibration of both base classifiers and conformal thresholds, a scenario addressed by adaptive conformal inference~\cite{gibbs2021adaptive}.

\textbf{Construct.}
Singleton accuracy measures the automated component of the framework but does not capture developer effort savings for non-singleton predictions. A user study evaluating actual triage time reduction is needed.

% VIII. CONCLUSION
% ============================================================
\section{Conclusion}\label{sec:conclusion}

We applied conformal prediction to multi-class software engineering triage -- to our knowledge, the first such application -- producing prediction sets with formal coverage guarantees for 30-class bug component assignment. Our central finding is that single-model confidence is insufficient for reliability partitioning in bug triage: within identical confidence bins, cross-model agreement reveals 43--69$\pp$ accuracy gaps on Eclipse and 12--56$\pp$ on Mozilla Core. MACCP, instantiating Mondrian conformal prediction with this binary agreement signal, achieves 61.2\% singleton sets at 90\% accuracy on Eclipse's agree partition (46.7\% of bugs) and 14.1\% singletons at 99.2\% accuracy on Mozilla Core's agree partition (30.9\% of bugs, $\alpha{=}0.20$).

Architectural diversity in deployed classifiers provides a conditioning signal for uncertainty quantification that comes at minimal additional cost. Where teams already run multiple models, cross-model agreement partitions predictions into reliability tiers with formal per-group coverage guarantees --- no retraining required.

\textbf{Limitations.} MACCP requires two competent classifiers with diverse feature representations. When one model is too weak, the agreement signal degrades. The temporal exchangeability violation means coverage guarantees are empirical, not theoretical, in our setting.

\textbf{Future work.}
(1)~Retrieval-augmented LLM explanations for the disagree group, surfacing similar historical bugs to help developers resolve ambiguous cases.
(2)~User study with developers measuring actual triage time reduction.
(3)~Extension to developer assignment (who should fix this?) rather than component assignment (which team?).
(4)~Online MACCP with adaptive conformal inference~\cite{gibbs2021adaptive} for continuous recalibration as component vocabularies evolve.
(5)~Investigation of score aggregation methods~\cite{alami2025sacp,ochoarivera2025csa} as alternatives to binary agreement partitioning.
(6)~Replacing DeBERTa with a large language model as the text classifier. MACCP accepts any classifier's probability distribution, and the cross-model agreement signal between an LLM and a metadata-based tree model may exhibit different characteristics --- LLMs with broader world knowledge could reduce the temporal vocabulary drift that drives much of the current disagreement, altering the agree/disagree distribution in ways that merit separate investigation.
